All public exports come from `@memoir/tree`. Import styles separately from `@memoir/tree/styles.css`.

```tsx
import { FamilyTree, OrgChart, org, rel } from "@memoir/tree";
import type { FamilyTreeProps, FamilyRelationship, FamilyTreeSpacing, OrgChartProps, TreeApi, TreeViewport } from "@memoir/tree";
import "@memoir/tree/styles.css";
```

## Family Tree

- `FamilyTree`
- `DefaultFamilyCard`
- `StyledFamilyCard`
- `rel`
- `createFamilyIndex(people, relationships)`
- `collectFamilyNeighborhood(index, subject, limits?)`
- `defaultFamilyNeighborhoodLimits`
- `buildFamilyTreeLayout(input)`

Use `FamilyTree` for normal app rendering. Use the pure helpers when you need to test relationship logic, precompute a layout, or build a custom renderer with the lower-level primitives.

```ts
type FamilyTreeProps<Person, CardExtraProps extends object = Record<string, never>> = {
  people?: Record<string, Person>;
  subject?: string;
  relationships: FamilyRelationship[];
  profiles?: Record<string, Person>;
  rootProfileId?: string;
  ariaLabel?: string;
  card?: React.ComponentType<FamilyCardProps<Person> & CardExtraProps>;
  cardProps?: CardExtraProps | ((person: Person, props: FamilyCardProps<Person>) => CardExtraProps);
  renderProfileCard?: (profile: Person, props: FamilyCardProps<Person>) => React.JSX.Element;
  className?: string;
  style?: React.CSSProperties;
  cardClassName?: string;
  edgeClassName?: string;
  interactionMode?: "pan" | "scroll" | "none";
  lineShape?: "orthogonal" | "curved";
  viewport?: TreeViewport;
  defaultViewport?: Partial<TreeViewport>;
  onViewportChange?: (viewport: TreeViewport) => void;
  treeApiRef?: React.Ref<TreeApi>;
  limits?: Partial<FamilyNeighborhoodLimits>;
  spacing?: Partial<FamilyTreeSpacing>;
  theme?: "memoir" | "system";
  getPersonLabel?: (person: Person, personId: string) => string;
  selected?: string;
  collapsed?: string[];
  readOnly?: boolean;
  onPersonClick?: (person: Person, personId: string) => void;
  onSelectProfile?: (person: Person, personId: string) => void;
  onAddRelationship?: (person: Person, personId: string) => void;
};
```

Prefer `people`, `subject`, `card`, and `onPersonClick` in new code. Compatibility aliases remain supported: `profiles`, `rootProfileId`, `renderProfileCard`, and `onSelectProfile`.

## FamilyTree Actions

These are the user-facing callbacks and imperative actions:

| Name | Purpose |
| --- | --- |
| `onPersonClick(person, personId)` | Fires when an interactive card is clicked or activated with Enter/Space. Prefer this in new code. |
| `onSelectProfile(person, personId)` | Memoir-shaped alias for `onPersonClick`. |
| `onAddRelationship(person, personId)` | Passed into cards so apps can open an “add relative” flow. The library does not mutate data. |
| `onViewportChange(viewport)` | Fires when pan position changes. Use it with controlled `viewport`. |
| `treeApiRef.current.centerPerson(personId)` | Scrolls a rendered person card toward the center of the viewport. |
| `treeApiRef.current.fitToSubject()` | Centers the current `subject`. |
| `treeApiRef.current.resetViewport()` | Returns to `defaultViewport`. |

## FamilyTree Props

| Name | Purpose |
| --- | --- |
| `people` | App-owned person records keyed by ID. Preferred name. |
| `subject` | Person ID at the center of the family neighborhood. Preferred name. |
| `relationships` | Flat parentage, partnership, and guardianship facts. |
| `profiles` | Compatibility alias for `people`. |
| `rootProfileId` | Compatibility alias for `subject`. |
| `card` | Custom React card component. Preferred path for app-owned card UI. |
| `cardProps` | Extra typed props passed into `card`; can be an object or function. |
| `renderProfileCard` | Compatibility/render-function path for custom cards. |
| `ariaLabel` | Accessible label for the tree surface. |
| `getPersonLabel` | Builds readable card labels for assistive technology. |
| `selected` | App-owned selected person ID. |
| `collapsed` | Person IDs whose deeper ancestor/descendant rows should be hidden. |
| `readOnly` | Disables add-relationship handlers passed to cards. |
| `viewport` | Controlled pan state: `{ x, y }`. |
| `defaultViewport` | Initial uncontrolled pan state. |
| `interactionMode` | `"pan"`, `"scroll"`, or `"none"`. |
| `limits` | Explicit visible-relative caps; use `null` to disable a cap. |
| `spacing` | Row, column, and padding spacing overrides. |
| `lineShape` | `"orthogonal"` or `"curved"` SVG edge routing. |
| `theme` | `"memoir"` or `"system"`. Use CSS variables/classes for custom colors. |
| `className` / `style` | Applied to the tree surface. |
| `cardClassName` / `edgeClassName` | Applied to cards and edge paths. |

## Relationship Helpers

- `rel.parents(child, parents, options?)`
- `rel.children(parents, children, options?)`
- `rel.partner(a, b, options?)`
- `rel.guardians(child, guardians, options?)`

```ts
rel.parents("alex", ["morgan", "casey"], {
  relation: "biological",
  status: "current",
  order: 0,
});

rel.partner("alex", "jordan", {
  relation: "spouse",
  status: "current",
});
```

Relationship option types:

- `ParentageRelation`: `biological`, `adoptive`, `step`, `foster`, `unknown`.
- `PartnershipRelation`: `spouse`, `partner`, `coparent`, `unknown`.
- `GuardianshipRelation`: `guardian`, `foster`, `unknown`.
- `FamilyRelationshipStatus`: `current`, `former`, `divorced`, `separated`, `unknown`.

## Org Chart

- `OrgChart`
- `DefaultOrgCard`
- `org`
- `createOrgChartIndex(people, relationships)`
- `collectOrgChartSubtree(index, root, options?)`
- `buildOrgChartLayout(input)`

Use `OrgChart` for people-based manager/report hierarchies:

```tsx
const relationships = [
  org.reports("ceo", ["vp-eng", "vp-sales"]),
];

<OrgChart
  people={people}
  root="ceo"
  relationships={relationships}
  renderCard={({ person, rootProps }) => (
    <article {...rootProps}>{person.name}</article>
  )}
/>;
```

`org.reports(managerId, reportIds, options?)` creates flat reporting facts. Org charts do not use family labels like `parent` or `sibling`; cards receive `depth` and `parentId`.

## Shared Surface

- `TreeSurface`
- `TreeProvider`
- `TreeCanvas`
- `TreeEdges`
- `TreeNodeLayer`
- `useTreeLayout()`
- `TreeInteractionMode`
- `TreeViewport`
- `TreeApi`
- `TreeLineShape`
- `TreeViewportProps`
- stylesheet from `@memoir/tree/styles.css`

Use primitives when the default wrapper is not enough but you still want package-owned layout, measurement, edges, and panning:

```tsx
<TreeProvider type="family" people={people} relationships={relationships} subject="alex">
  <TreeCanvas ariaLabel="Alex's family tree">
    <TreeEdges />
    <TreeNodeLayer card={ProfileCard} />
  </TreeCanvas>
</TreeProvider>
```

Primitive responsibilities:

| Primitive | Purpose |
| --- | --- |
| `TreeProvider` | Creates family layout context from people, relationships, selection, collapse state, labels, limits, and spacing. |
| `TreeCanvas` | Renders the viewport surface, pan behavior, and canvas container. |
| `TreeEdges` | Renders SVG relationship edges from the computed layout. |
| `TreeNodeLayer` | Renders measured card nodes with accessibility and data attributes. |
| `TreeSurface` | Lower-level pan surface used by `TreeCanvas`. |
| `useTreeLayout()` | Reads computed cards, edges, bounds, and tree context for custom renderers. |
