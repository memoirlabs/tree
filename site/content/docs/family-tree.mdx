---
title: FamilyTree
description: Render a subject-centered family neighborhood from graph or simple relationship facts.
---

`FamilyTree` renders one person's readable family neighborhood from app-owned people and family graph facts. Simple relationship helpers are still available for small trees and examples.

It handles measured cards, SVG relationship edges, panning, accessible card props, custom React cards, and the tiny stylesheet.

## Simple Mode

Simple mode uses `rel` helpers. It is compact, but graph mode is preferred for real family apps because parentage is per link and children can belong to explicit partnership groups.

```tsx
import { FamilyTree, rel } from "@memoir/tree";
import "@memoir/tree/styles.css";

const people = {
  alex: { id: "alex", name: "Alex" },
  morgan: { id: "morgan", name: "Morgan" },
  casey: { id: "casey", name: "Casey" },
  jordan: { id: "jordan", name: "Jordan" },
  riley: { id: "riley", name: "Riley" },
};

const relationships = [
  rel.parents("alex", ["morgan", "casey"]),
  rel.partner("alex", "jordan", { relation: "spouse" }),
  rel.children(["alex", "jordan"], ["riley"]),
];

export function FamilyPanel() {
  return <FamilyTree people={people} subject="alex" relationships={relationships} />;
}
```

You can omit `card` to use `DefaultFamilyCard`.

## Graph Mode

```tsx
import { FamilyTree, type FamilyGraph } from "@memoir/tree";

const graph: FamilyGraph<Person> = {
  people,
  subject: "riley",
  partnershipGroups: [
    { id: "alex-jordan", partners: ["alex", "jordan"], relation: "spouse" },
    { id: "alex-morgan", partners: ["alex", "morgan"], relation: "coparent", status: "former" },
  ],
  parentChildLinks: [
    { id: "alex-riley", groupId: "alex-jordan", parentId: "alex", childId: "riley", relation: "biological" },
    { id: "jordan-riley", groupId: "alex-jordan", parentId: "jordan", childId: "riley", relation: "step" },
    { id: "alex-casey", groupId: "alex-morgan", parentId: "alex", childId: "casey" },
    { id: "morgan-casey", groupId: "alex-morgan", parentId: "morgan", childId: "casey" },
  ],
  guardianshipLinks: [
    { id: "morgan-riley-guardian", guardianId: "morgan", childId: "riley", relation: "guardian" },
  ],
};

<FamilyTree graph={graph} />;
```

Tree renders and lays out the graph. Your app still owns persistence, permissions, editing, routing, invites, and data validation.

## Custom Card

```tsx
import type { FamilyCardProps } from "@memoir/tree";

type Person = (typeof people)[string];

function ProfileCard({ focused, person, relation, ...rootProps }: FamilyCardProps<Person>) {
  return (
    <article {...rootProps}>
      <strong>{person.name}</strong>
      <small>{relation.label}</small>
    </article>
  );
}

<FamilyTree people={people} subject="alex" relationships={relationships} card={ProfileCard} />;
```

The card receives accessibility props, keyboard handlers, stable `data-*` attributes, relationship metadata, selection state, and the original person record. Spread the remaining props onto the card root.

## Relationship Helpers

```ts
rel.parents("alex", ["morgan", "casey"], {
  relation: "biological",
  status: "current",
});

rel.partner("alex", "jordan", {
  relation: "spouse",
  status: "current",
});

rel.guardians("riley", ["morgan"]);
```

These helpers are simple-mode helpers. `FamilyTree` also accepts graph mode for partnership groups, per-parent lineage, multiple unions, and guardianship. App-specific data like permissions, profile drawers, images, and actions belongs on your person records or typed `cardProps`.

## Common Options

```tsx
<FamilyTree
  people={people}
  subject="alex"
  relationships={relationships}
  selected={selectedPersonId}
  collapsed={["riley"]}
  limits={{ children: null, siblings: 12 }}
  onPersonClick={(_person, personId) => setSelectedPersonId(personId)}
/>
```

Use `selected` for app-owned selection, `collapsed` to hide deeper ancestor or descendant rows, and `limits` to control visible-relative caps.

Use `initialViewport="subject"` to center the subject after cards are measured. For viewport controls, pass `treeApiRef`. Available helpers are `centerPerson(personId)`, `fitToSubject()`, and `resetViewport()`.
