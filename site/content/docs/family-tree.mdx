`FamilyTree` renders a subject-centered family neighborhood from app-owned person data and plain relationship facts.

## Props

```ts
type FamilyTreeProps<Person> = {
  rootProfileId?: string;
  subject?: string;
  profiles?: Record<string, Person>;
  people?: Record<string, Person>;
  relationships: FamilyRelationship[];
  card?: React.ComponentType<FamilyCardProps<Person>>;
  renderProfileCard?: (profile: Person, props: FamilyCardProps<Person>) => React.JSX.Element;
  className?: string;
  style?: React.CSSProperties;
  cardClassName?: string;
  edgeClassName?: string;
  interactionMode?: "pan" | "scroll" | "none";
  lineShape?: "orthogonal" | "curved";
  zoom?: number;
  defaultZoom?: number;
  minZoom?: number;
  maxZoom?: number;
  onZoomChange?: (zoom: number) => void;
  spacing?: Partial<FamilyTreeSpacing>;
  theme?: "memoir" | "system" | TreeTheme;
  selected?: string;
  collapsed?: string[];
  readOnly?: boolean;
  onSelectProfile?: (person: Person, personId: string) => void;
  onAddRelationship?: (person: Person, personId: string) => void;
  onPersonClick?: (person: Person, personId: string) => void;
};
```

The Memoir-shaped names are `profiles`, `rootProfileId`, `renderProfileCard`, and `onSelectProfile`. The older generic names, `people`, `subject`, `card`, and `onPersonClick`, still work for compatibility.

The library does not require `name`, `avatarUrl`, profile fields, or nested relationship arrays. It only needs stable profile IDs.

By default, `FamilyTree` is self-contained: it fills the parent width, inherits parent height when one exists, falls back to a usable minimum height, clips overflow, and lets users drag-pan the tree horizontally and vertically. Host apps can pass `zoom`, `style`, `className`, or `interactionMode` when they want to override those defaults.

## Controlled Selection

Use `selected` and `onSelectProfile` when the host app owns profile drawers, route state, or inspectors:

```tsx
function FamilyPanel() {
  const [selectedProfileId, setSelectedProfileId] = useState("alex");

  return (
    <FamilyTree
      profiles={profiles}
      relationships={relationships}
      rootProfileId="alex"
      selected={selectedProfileId}
      onSelectProfile={(_profile, profileId) => setSelectedProfileId(profileId)}
      renderProfileCard={renderProfileCard}
    />
  );
}
```

`onPersonClick` is still available as the older generic name. Prefer `onSelectProfile` in new Memoir-style integrations.

## Relationship Facts

<div className="docs-api-band">
  <h3>Relationships are facts, not nested UI state.</h3>
  <div className="docs-copy">Store parentage, partnership, and guardianship as flat rows. The subject-relative labels are computed at render time.</div>
</div>

```ts
import { rel } from "@memoir/tree";

const relationships = [
  rel.parents("alex", ["morgan", "casey"]),
  rel.partner("alex", "jordan", { relation: "spouse" }),
  rel.children(["alex", "jordan"], ["riley", "quinn"]),
  rel.guardians("riley", ["morgan"]),
];
```

The helpers produce plain rows:

- `parentage`: parent/child facts.
- `partnership`: spouse, partner, or coparent facts.
- `guardianship`: guardian/child facts.

Relationship options let you preserve real family nuance without giving the renderer business rules:

```ts
const relationships = [
  rel.parents("quinn", ["alex", "jordan"], { relation: "adoptive" }),
  rel.partner("alex", "drew", { relation: "coparent", status: "former" }),
  rel.guardians("quinn", ["robin"], { relation: "guardian" }),
];
```

Supported parentage relations are `biological`, `adoptive`, `step`, `foster`, and `unknown`. Supported partnership relations are `spouse`, `partner`, `coparent`, and `unknown`. Supported statuses are `current`, `former`, `divorced`, `separated`, and `unknown`.

## Card Contract

If you omit `renderProfileCard`, `FamilyTree` uses `DefaultFamilyCard`, which displays `name`, `label`, `profile.display`, or the profile ID.

Your custom card receives the profile object, computed relation metadata, selected state, collapse state, and edit hooks:

```tsx
import type { FamilyCardProps } from "@memoir/tree";

function ProfileCard({
  collapsed,
  focused: _focused,
  onAddRelationship,
  person,
  personId,
  readOnly,
  relation,
  selected,
  ...props
}: FamilyCardProps<Person>) {
  return (
    <article {...props} className="memoir-profile-card">
      <strong>{person.profile.display}</strong>
      <small>{relation.label}</small>
      {selected ? <span>Selected</span> : null}
      {collapsed ? <span>Collapsed</span> : null}
      {!readOnly ? (
        <button type="button" onClick={() => onAddRelationship?.(person, personId)}>
          Add relative
        </button>
      ) : null}
    </article>
  );
}
```

Computed relation labels include `self`, `parent`, `grandparent`, `child`, `grandchild`, `sibling`, `half-sibling`, `partner`, `coparent`, `guardian`, and `unknown`.

## Layout

The default layout shows a subject-centered family window:

- generation `-2`: grandparents
- generation `-1`: parents and guardians
- generation `0`: siblings, half-siblings, subject, partners, coparents
- generation `1`: children
- generation `2`: grandchildren

Cards are measured with `ResizeObserver`. The first pass uses a fallback size, then layout recalculates with the actual rendered card dimensions.

The root element owns the viewport behavior. `interactionMode="pan"` uses a clipped drag-panning viewport, `interactionMode="scroll"` exposes native scrollbars, and `interactionMode="none"` clips without user navigation.

Use `collapsed` to hide deeper ancestor or descendant rows while keeping the collapsed profile visible:

```tsx
<FamilyTree
  profiles={profiles}
  relationships={relationships}
  rootProfileId="alex"
  collapsed={["riley"]}
/>
```

Use `zoom` for a controlled viewport or `defaultZoom` for an uncontrolled starting value:

```tsx
<FamilyTree
  profiles={profiles}
  relationships={relationships}
  rootProfileId="alex"
  zoom={zoom}
  minZoom={0.6}
  maxZoom={2}
  onZoomChange={setZoom}
/>
```
