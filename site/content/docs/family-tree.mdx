`FamilyTree` renders a subject-centered family neighborhood from app-owned person data and plain relationship facts.

## Props

```ts
type FamilyTreeProps<Person, CardExtraProps extends object = Record<string, never>> = {
  subject?: string;
  people?: Record<string, Person>;
  rootProfileId?: string;
  profiles?: Record<string, Person>;
  relationships: FamilyRelationship[];
  ariaLabel?: string;
  card?: React.ComponentType<FamilyCardProps<Person> & CardExtraProps>;
  cardProps?: CardExtraProps | ((person: Person, props: FamilyCardProps<Person>) => CardExtraProps);
  renderProfileCard?: (profile: Person, props: FamilyCardProps<Person>) => React.JSX.Element;
  className?: string;
  style?: React.CSSProperties;
  cardClassName?: string;
  edgeClassName?: string;
  interactionMode?: "pan" | "scroll" | "none";
  lineShape?: "orthogonal" | "curved";
  zoom?: number;
  defaultZoom?: number;
  viewport?: TreeViewport;
  defaultViewport?: Partial<TreeViewport>;
  onViewportChange?: (viewport: TreeViewport) => void;
  minZoom?: number;
  maxZoom?: number;
  onZoomChange?: (zoom: number) => void;
  spacing?: Partial<FamilyTreeSpacing>;
  limits?: Partial<FamilyNeighborhoodLimits>;
  theme?: "memoir" | "system" | TreeTheme;
  treeApiRef?: React.Ref<TreeApi>;
  getPersonLabel?: (person: Person, personId: string) => string;
  selected?: string;
  collapsed?: string[];
  readOnly?: boolean;
  onPersonClick?: (person: Person, personId: string) => void;
  onSelectProfile?: (person: Person, personId: string) => void;
  onAddRelationship?: (person: Person, personId: string) => void;
};
```

Prefer the clean public names in new integrations: `people`, `subject`, `card`, and `onPersonClick`. The Memoir-shaped names `profiles`, `rootProfileId`, `renderProfileCard`, and `onSelectProfile` remain supported aliases for app compatibility.

The library does not require `name`, `avatarUrl`, profile fields, or nested relationship arrays. It only needs stable profile IDs.

By default, `FamilyTree` is self-contained: it fills the parent width, inherits parent height when one exists, falls back to a usable minimum height, clips overflow, and lets users drag-pan the tree horizontally and vertically. Host apps can pass `zoom`, `style`, `className`, or `interactionMode` when they want to override those defaults.

## Controlled Selection

Use `selected` and `onPersonClick` when the host app owns profile drawers, route state, or inspectors:

```tsx
function FamilyPanel() {
  const [selectedProfileId, setSelectedProfileId] = useState("alex");

  return (
    <FamilyTree
      profiles={profiles}
      relationships={relationships}
      rootProfileId="alex"
      selected={selectedProfileId}
      onPersonClick={(_profile, profileId) => setSelectedProfileId(profileId)}
      renderProfileCard={renderProfileCard}
    />
  );
}
```

`onSelectProfile` remains available as the Memoir-shaped alias.

## Actions

`FamilyTree` never mutates your data. It only calls your handlers:

- `onPersonClick` opens selection, drawers, routes, or inspectors.
- `onSelectProfile` is the Memoir-shaped alias for selection.
- `onAddRelationship` is passed into cards so the app can open an add-relative flow.
- `onViewportChange` reports pan and zoom.
- `onZoomChange` reports only zoom.

For viewport buttons, use `treeApiRef`:

```tsx
const treeApiRef = useRef<TreeApi | null>(null);

<button type="button" onClick={() => treeApiRef.current?.fitToSubject()}>
  Center subject
</button>

<button type="button" onClick={() => treeApiRef.current?.resetViewport()}>
  Reset view
</button>

<FamilyTree
  treeApiRef={treeApiRef}
  people={people}
  relationships={relationships}
  subject="alex"
/>
```

## Relationship Facts

Store parentage, partnership, and guardianship as flat facts. Subject-relative labels are computed at render time.

```ts
import { rel } from "@memoir/tree";

const relationships = [
  rel.parents("alex", ["morgan", "casey"]),
  rel.partner("alex", "jordan", { relation: "spouse" }),
  rel.children(["alex", "jordan"], ["riley", "quinn"]),
  rel.guardians("riley", ["morgan"]),
];
```

The helpers produce plain rows:

- `parentage`: parent/child facts.
- `partnership`: spouse, partner, or coparent facts.
- `guardianship`: guardian/child facts.

Relationship options let you preserve real family nuance without giving the renderer business rules:

```ts
const relationships = [
  rel.parents("quinn", ["alex", "jordan"], { relation: "adoptive" }),
  rel.partner("alex", "drew", { relation: "coparent", status: "former" }),
  rel.guardians("quinn", ["robin"], { relation: "guardian" }),
];
```

Supported parentage relations are `biological`, `adoptive`, `step`, `foster`, and `unknown`. Supported partnership relations are `spouse`, `partner`, `coparent`, and `unknown`. Supported statuses are `current`, `former`, `divorced`, `separated`, and `unknown`.

## Card Contract

If you omit `renderProfileCard`, `FamilyTree` uses `DefaultFamilyCard`, which displays `name`, `label`, `profile.display`, or the profile ID.

Your custom card receives the person object, computed relation metadata, selected/focused state, accessibility props, stable data attributes, collapse state, and edit hooks. Always spread `...props` onto the card root:

```tsx
import type { FamilyCardProps } from "@memoir/tree";

function ProfileCard({
  collapsed,
  focused: _focused,
  onAddRelationship,
  person,
  personId,
  readOnly,
  relation,
  selected,
  ...props
}: FamilyCardProps<Person>) {
  return (
    <article {...props} className="memoir-profile-card">
      <strong>{person.profile.display}</strong>
      <small>{relation.label}</small>
      {selected ? <span>Selected</span> : null}
      {collapsed ? <span>Collapsed</span> : null}
      {!readOnly ? (
        <button type="button" onClick={() => onAddRelationship?.(person, personId)}>
          Add relative
        </button>
      ) : null}
    </article>
  );
}
```

Computed relation labels include `self`, `parent`, `grandparent`, `child`, `grandchild`, `sibling`, `half-sibling`, `partner`, `coparent`, `guardian`, and `unknown`.

Pass app-owned props to a custom card with `cardProps`:

```tsx
type ProfileCardExtraProps = {
  canEdit: boolean;
  density: "compact" | "comfortable";
};

function ProfileCard({
  canEdit,
  density,
  person,
  ...props
}: FamilyCardProps<Person> & ProfileCardExtraProps) {
  return (
    <article {...props} data-density={density}>
      <strong>{person.profile.display}</strong>
      {canEdit ? <span>editable</span> : null}
    </article>
  );
}

<FamilyTree<Person, ProfileCardExtraProps>
  people={people}
  relationships={relationships}
  subject="alex"
  card={ProfileCard}
  cardProps={(person) => ({
    canEdit: Boolean(person.permissions?.canEdit),
    density: "compact",
  })}
/>
```

## Accessibility

`FamilyTree` labels the viewport and each interactive card. Use `ariaLabel` for the tree surface and `getPersonLabel` for readable card labels:

```tsx
<FamilyTree
  ariaLabel="Alex's family tree"
  getPersonLabel={(person) => person.profile.display}
  people={people}
  relationships={relationships}
  subject="alex"
  onPersonClick={openPerson}
/>
```

When a card is clickable, the tree passes `role="button"`, `tabIndex={0}`, keyboard activation handlers, `aria-selected`, `data-selected`, and `data-focused`.

## Layout

The default layout shows a subject-centered family window:

- generation `-2`: grandparents
- generation `-1`: parents and guardians
- generation `0`: siblings, half-siblings, subject, partners, coparents
- generation `1`: children
- generation `2`: grandchildren

Cards are measured with `ResizeObserver`. The first pass uses a fallback size, then layout recalculates with the actual rendered card dimensions.

The root element owns the viewport behavior. `interactionMode="pan"` uses a clipped drag-panning viewport, `interactionMode="scroll"` exposes native scrollbars, and `interactionMode="none"` clips without user navigation.

Use `collapsed` to hide deeper ancestor or descendant rows while keeping the collapsed profile visible:

```tsx
<FamilyTree
  profiles={profiles}
  relationships={relationships}
  rootProfileId="alex"
  collapsed={["riley"]}
/>
```

Use `viewport` for controlled pan and zoom, or `defaultViewport` for an uncontrolled starting position:

```tsx
<FamilyTree
  people={people}
  relationships={relationships}
  subject="alex"
  viewport={viewport}
  onViewportChange={setViewport}
  minZoom={0.6}
  maxZoom={2}
/>
```

For simple zoom-only integrations, `zoom`, `defaultZoom`, and `onZoomChange` remain available.

Available helpers are `centerPerson(personId)`, `fitToSubject()`, `resetViewport()`, and `zoomTo(zoom)`.

## Neighborhood Limits

The layout renders a subject-centered family window. Defaults keep large families readable, and `limits` makes those caps explicit:

```tsx
<FamilyTree
  people={people}
  relationships={relationships}
  subject="alex"
  limits={{
    siblings: 12,
    children: null,
    grandchildren: 16,
  }}
/>
```

Use `null` to disable a cap for a group. Defaults are exported as `defaultFamilyNeighborhoodLimits`.
