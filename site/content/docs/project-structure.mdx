You do not need to copy this library's source tree into your app. Install the package, keep your existing app structure, and add the tree wherever you render family context.

## What Your App Owns

`@memoir/tree` is a renderer. Your app should continue to own:

- Where people are stored.
- How relationships are created or edited.
- Permissions, auth, routing, drawers, and forms.
- The visual markup inside each person card.

The library only needs stable person IDs, a `people` map, relationship facts, and a subject ID.

## Recommended Shape

Put your data and relationship helpers wherever they already belong in your app. A simple setup can look like this:

```text
app/
  family/
    page.tsx
    profile-card.tsx
    family-data.ts
```

`family-data.ts` can export your people and relationship facts:

```ts
import { rel } from "@memoir/tree";

export const people = {
  alex: { id: "alex", name: "Alex" },
  morgan: { id: "morgan", name: "Morgan" },
  casey: { id: "casey", name: "Casey" },
};

export const relationships = [
  rel.parents("alex", ["morgan", "casey"]),
];
```

`profile-card.tsx` can render your app's card UI:

```tsx
import type { FamilyCardProps } from "@memoir/tree";

type Person = {
  id: string;
  name: string;
};

export function ProfileCard({ person, relation, ...props }: FamilyCardProps<Person>) {
  return (
    <article {...props}>
      <strong>{person.name}</strong>
      <small>{relation.label}</small>
    </article>
  );
}
```

Then render the tree on any page:

```tsx
import { FamilyTree } from "@memoir/tree";
import "@memoir/tree/styles.css";
import { people, relationships } from "./family-data";
import { ProfileCard } from "./profile-card";

export default function FamilyPage() {
  return (
    <FamilyTree
      people={people}
      relationships={relationships}
      subject="alex"
      card={ProfileCard}
    />
  );
}
```

## Integration Checklist

- Import `@memoir/tree/styles.css` once if you want the default Memoir skin.
- Pass `people`, `relationships`, and `subject` into `FamilyTree`.
- Use `card` when you want custom app-owned person cards.
- Use `selected` and `onPersonClick` when your app owns selection state.
- Use `onAddRelationship` to open your own add-relative flow.
