## Folder Responsibilities

- `src/family/indexing.ts`: Pure relationship fact indexing and subject-relative labels.
- `src/family/layout.ts`: Pure measured row placement.
- `src/family/org.ts`: Nested org chart normalization.
- `src/family/org-layout.ts`: Pure org chart measured row placement.
- `src/family/edge-routing.ts`: Pure SVG path generation.
- `src/family/theme.ts`: Theme presets and CSS variable mapping.
- `src/family/tree-dsl.ts`: Small diagram syntax for docs, demos, and tests.
- `src/family/use-card-measurements.ts`: DOM measurement hook.
- `src/family/FamilyTree.tsx`: Thin React renderer for cards, canvas, and edges.
- `src/family/TreePrimitives.tsx`: Composable provider, canvas, edge, and node layers.
- `src/styles.css`: Optional package stylesheet.
- `tests/`: Bun tests for helpers, family logic, and layout behavior.
- `site/`: Next.js app with the landing page, simple rendered demo, and Fumadocs docs.
- `site/app/(home)/page.tsx`: Landing page at `/`.
- `site/app/docs`: Fumadocs routes at `/docs`.
- `site/components/simple-family-tree-demo.tsx`: Static homepage demo that imports local library source.
- `site/content/docs`: Fumadocs MDX content.
- `dist/`: Generated package output.

## Public Entrypoints

- npm package entry: `src/index.ts` -> `dist/index.js`.

## Package Boundary

Only these files ship via npm:

- `dist`
- `public/logo-transparent.png`
- `README.md`
- `LICENSE`

Tests, scripts, demo code, and docs stay local.
