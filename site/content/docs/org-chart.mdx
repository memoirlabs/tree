---
title: Org Chart
description: "@memoir/tree is for family trees, not org charts."
---

`@memoir/tree` does not currently export an `OrgChart` component.

That is intentional. This package is scoped to subject-centered family relationships: parentage, partnership, guardianship, siblings, descendants, ancestors, and relationship-aware labels.

## Use FamilyTree For Families

Use `FamilyTree` when the data is family data:

```tsx
import { FamilyTree, rel } from "@memoir/tree";
import "@memoir/tree/styles.css";

const relationships = [
  rel.parents("alex", ["morgan", "casey"]),
  rel.partner("alex", "jordan"),
];
```

## Do Not Import OrgChart

```tsx
import { OrgChart } from "@memoir/tree";
```

That import is not supported.

## Why Not Force It

Org charts have different semantics from family trees:

- Managers and direct reports are not parentage.
- Dotted-line reporting is not guardianship.
- Teams, job titles, vacant roles, and reporting permissions need their own model.
- Org chart labels should not be computed as family labels.

If your data is not family data, use a tool designed for that model instead of reshaping it into family relationships.
