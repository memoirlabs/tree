title: Integration
description: Build app features around @memoir/tree without moving your data model.
---

`@memoir/tree` is meant to sit inside your existing React app. It does not bring a data layer, editor, router, auth system, or form flow.

## App Responsibilities

Keep these parts in your application:

- Person records and profile fields.
- Relationship creation, validation, and persistence.
- Permissions for viewing or editing family information.
- Selection state, side panels, forms, routes, and analytics.
- Custom card markup and app-specific actions.

The package handles rendering: layout, measured cards, SVG edges, panning, subject-relative labels, and accessibility props.

## Data Flow

Fetch or compute your people and relationships before rendering the tree:

```tsx
import { FamilyTree } from "@memoir/tree";
import "@memoir/tree/styles.css";

export function FamilyPanel({ family }: { family: FamilyData }) {
  return (
    <FamilyTree
      people={family.people}
      relationships={family.relationships}
      subject={family.subjectId}
      selected={family.selectedId}
      onPersonClick={family.openProfile}
    />
  );
}
```

The tree never writes back to your data. When a user clicks a card or asks to add a relative, call your own app code.

## Editing Flows

Use callbacks to connect tree interactions to your UI:

```tsx
<FamilyTree
  people={people}
  relationships={relationships}
  subject="alex"
  selected={selectedPersonId}
  onPersonClick={(_person, personId) => setSelectedPersonId(personId)}
  onAddRelationship={(_person, personId) => openAddRelativeDialog(personId)}
/>
```

Set `readOnly` when cards should not receive add-relationship actions:

```tsx
<FamilyTree
  people={people}
  relationships={relationships}
  subject="alex"
  readOnly={!canEditFamily}
/>
```

## Custom Cards

Custom cards are regular React components. Spread the props onto the root element so the tree can attach accessibility handlers and stable data attributes:

```tsx
import type { FamilyCardProps } from "@memoir/tree";

type Person = {
  id: string;
  name: string;
  avatarUrl?: string;
};

export function ProfileCard({ focused, person, relation, ...props }: FamilyCardProps<Person>) {
  return (
    <article {...props}>
      {person.avatarUrl ? <img src={person.avatarUrl} alt="" /> : null}
      <strong>{person.name}</strong>
      <small>{focused && relation.label === "self" ? "root node" : relation.label}</small>
    </article>
  );
}
```

## Practical Checks

When integrating the tree into an app, check:

- Cards render with the expected names and labels.
- Clicking a card opens your app's selection UI.
- Keyboard activation works when `onPersonClick` is provided.
- Large families stay readable with `limits`.
- Custom card roots spread `...props`.

For the full prop surface, see [FamilyTree](./family-tree) and [API](./api).
