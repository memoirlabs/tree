`@memoir/tree` ships a small optional stylesheet, but the styling model is deliberately ordinary React and CSS. The renderer gives you stable attributes and CSS variables; your app supplies the final visual language.

## Import Styles

Import the stylesheet once if you want the default selectors and theme variables:

```tsx
import "@memoir/tree/styles.css";
```

The stylesheet is optional. Without it, the React components still render; you just own more of the CSS yourself.

## Flat Black-Shadow Style

Use this as the recommended starting point for flat product UIs:

```css
.memoir-tree {
  --tree-surface-bg: #fffdf4;
  --tree-canvas-bg: #f4efdc;
  --tree-card-bg: #fffdf4;
  --tree-card-fg: #030201;
  --tree-card-border: #030201;
  --tree-card-selected-bg: #ec5a44;
  --tree-card-shadow: 6px 6px 0 #030201;
  --tree-edge: #3b342e;
  --tree-edge-width: 2;
  --tree-card-radius: 0;
  --tree-profile-radius: 0;
}

.memoir-profile-card {
  display: grid;
  min-width: 128px;
  justify-items: center;
  gap: 5px;
  padding: 10px 12px;
  border: 2px solid #030201;
  background: #fffdf4;
  box-shadow: 6px 6px 0 #030201;
  color: #030201;
  text-align: center;
}

.memoir-profile-card[aria-selected="true"] {
  background: #ec5a44;
}
```

Then pass the surface class and use a card component that forwards only DOM-safe props:

```tsx
function ProfileCard({
  collapsed: _collapsed,
  focused: _focused,
  onAddRelationship: _onAddRelationship,
  person,
  personId: _personId,
  readOnly: _readOnly,
  relation,
  selected: _selected,
  ...props
}: FamilyCardProps<Profile>) {
  return (
    <article {...props} className="memoir-profile-card">
      <strong>{person.name}</strong>
      <small>{relation.label}</small>
    </article>
  );
}

<FamilyTree
  className="memoir-tree"
  profiles={profiles}
  relationships={relationships}
  rootProfileId={profileId}
  renderProfileCard={(_profile, props) => <ProfileCard {...props} />}
/>
```

## Theme Prop

For small changes, use the `theme` prop instead of writing CSS variables manually:

```tsx
<FamilyTree
  profiles={profiles}
  relationships={relationships}
  rootProfileId="alex"
  theme={{
    surfaceBackground: "#fffdf4",
    canvasBackground: "#f4efdc",
    cardBackground: "#fffdf4",
    cardBorder: "#030201",
    cardRadius: 0,
    cardShadow: "6px 6px 0 #030201",
    edge: "#3b342e",
    edgeWidth: 2,
    accent: "#ec5a44",
  }}
/>
```

Use `theme="memoir"` for the package default or `theme="system"` for neutral system colors.

## Stable Selectors

The family renderer emits these selectors:

- `data-tree-surface`: outer scroll/pan surface.
- `data-tree-canvas`: measured canvas that contains edges and cards.
- `data-family-tree`: family tree canvas marker.
- `data-family-card-positioner`: absolute-positioned wrapper for a card.
- `data-family-card`: card props forwarded to your custom card.
- `data-person-id`: current profile ID.
- `data-relation`: computed relation label like `self`, `parent`, or `half-sibling`.
- `data-generation`: generation relative to the root profile.
- `data-side`: `ancestor`, `descendant`, `sibling`, `partner`, or `self`.
- `data-family-edge`: SVG edge path.
- `data-edge-kind`: relationship kind like `biological`, `adoptive`, `spouse`, or `guardian`.
- `data-edge-status`: relationship status like `current`, `former`, or `divorced`.

Example:

```css
[data-family-card][data-relation="half-sibling"] {
  background: #ffea00;
}

[data-family-edge][data-edge-kind="adoptive"],
[data-family-edge][data-edge-status="former"] {
  stroke-dasharray: 4 4;
}
```

## Card Responsibilities

Your card receives normal HTML props plus tree metadata. Always spread the props onto the root element so selection, data attributes, click handlers, and accessibility metadata reach the DOM:

```tsx
function ProfileCard({
  collapsed,
  focused: _focused,
  person,
  personId,
  readOnly,
  relation,
  selected,
  onAddRelationship,
  ...props
}: FamilyCardProps<Profile>) {
  return (
    <article {...props} className="memoir-profile-card">
      <strong>{person.name}</strong>
      <small>{selected ? "selected" : relation.label}</small>
      {!readOnly ? (
        <button type="button" onClick={() => onAddRelationship?.(person, personId)}>
          Add relative
        </button>
      ) : null}
      {collapsed ? <span>Collapsed</span> : null}
    </article>
  );
}
```

If you do not want metadata props like `personId`, `relation`, or `readOnly` to appear on a DOM element, destructure them before spreading, as shown above.
