`@memoir/tree` is a lightweight React renderer for family trees and org charts. Your app owns the people, permissions, editing flows, and database writes. The package handles measured layout, SVG relationship edges, panning, card placement, accessibility props, and a tiny CSS-variable skin.

<Image className="docs-intro-logo" src={logo} alt="Memoir Tree" />

The package is not a database or editor in a box. It keeps persistence, routing, auth, permissions, and data fetching in the host application. It focuses on:

- Normalizing relationship facts.
- Computing subject-relative family labels.
- Computing rooted org chart hierarchy.
- Measuring custom cards.
- Rendering automatic layouts, SVG edges, panning, and a small viewport API.

## Install

```bash
bun add @memoir/tree
```

Import the stylesheet once for the intended Memoir skin. It is framework-free CSS built from variables and stable selectors:

```tsx
import "@memoir/tree/styles.css";
```

## Minimal Family Tree

This example uses the clean API: `people`, `relationships`, `subject`, and `card`.

```tsx
import { FamilyTree, rel } from "@memoir/tree";
import type { FamilyCardProps } from "@memoir/tree";
import "@memoir/tree/styles.css";

const people = {
  alex: { id: "alex", name: "Alex" },
  morgan: { id: "morgan", name: "Morgan" },
  casey: { id: "casey", name: "Casey" },
  jordan: { id: "jordan", name: "Jordan" },
  riley: { id: "riley", name: "Riley" },
};

const relationships = [
  rel.parents("alex", ["morgan", "casey"]),
  rel.partner("alex", "jordan"),
  rel.children(["alex", "jordan"], ["riley"]),
];

type Person = (typeof people)[string];

function ProfileCard({
  collapsed: _collapsed,
  focused,
  onAddRelationship: _onAddRelationship,
  person,
  personId: _personId,
  readOnly: _readOnly,
  relation,
  ...props
}: FamilyCardProps<Person>) {
  return (
    <article {...props} className="memoir-profile-card">
      <strong>{person.name}</strong>
      <small>{focused && relation.label === "self" ? "root node" : relation.label}</small>
    </article>
  );
}

export function Page() {
  return (
    <FamilyTree
      people={people}
      relationships={relationships}
      subject="alex"
      card={ProfileCard}
    />
  );
}
```

You can also omit `card` to use the built-in default card:

```tsx
export function Page() {
  return <FamilyTree people={people} subject="alex" relationships={relationships} />;
}
```

## Minimal Org Chart

```tsx
import { OrgChart, org } from "@memoir/tree";
import "@memoir/tree/styles.css";

const people = {
  ceo: { id: "ceo", name: "Casey", title: "CEO" },
  eng: { id: "eng", name: "Morgan", title: "VP Engineering" },
  design: { id: "design", name: "Riley", title: "Design Lead" },
};

const relationships = [
  org.reports("ceo", ["eng", "design"]),
];

export function Page() {
  return <OrgChart people={people} root="ceo" relationships={relationships} />;
}
```

For custom org cards, use `renderCard`. The `rootProps` object is the only thing that needs to be spread onto your card root:

```tsx
<OrgChart
  people={people}
  root="ceo"
  relationships={relationships}
  renderCard={({ person, depth, rootProps }) => (
    <article {...rootProps}>
      <strong>{person.name}</strong>
      <small>{person.title ?? `level ${depth}`}</small>
    </article>
  )}
/>
```

`profiles`, `rootProfileId`, `renderProfileCard`, and `onSelectProfile` remain supported Memoir-shaped aliases for app compatibility.
