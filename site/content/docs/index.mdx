`@memoir/tree` is a controlled React renderer for family trees. Your app owns the profiles, permissions, editing flows, and database writes. The package handles measured layout, SVG relationship edges, panning, zooming, and card placement.

The package is intentionally not a genealogy app in a box. It keeps persistence, routing, styling, auth, permissions, and data fetching in the host application. It focuses on:

- Normalizing relationship facts.
- Computing subject-relative family labels.
- Measuring custom cards.
- Rendering automatic layouts, SVG edges, pan, and zoom.

## Install

```bash
bun add @memoir/tree
```

Import the optional stylesheet once in your app if you want the default Memoir-flavored CSS variables and stable selector styles:

```tsx
import "@memoir/tree/styles.css";
```

## Minimal Family Tree

This example uses the Memoir-shaped API: `profiles`, `relationships`, `rootProfileId`, and `renderProfileCard`.

```tsx
import { FamilyTree, rel } from "@memoir/tree";
import type { FamilyCardProps } from "@memoir/tree";
import "@memoir/tree/styles.css";

const profiles = {
  alex: { id: "alex", name: "Alex" },
  morgan: { id: "morgan", name: "Morgan" },
  casey: { id: "casey", name: "Casey" },
  jordan: { id: "jordan", name: "Jordan" },
  riley: { id: "riley", name: "Riley" },
};

const relationships = [
  rel.parents("alex", ["morgan", "casey"]),
  rel.partner("alex", "jordan"),
  rel.children(["alex", "jordan"], ["riley"]),
];

type Profile = (typeof profiles)[string];

function ProfileCard({
  collapsed: _collapsed,
  focused: _focused,
  onAddRelationship: _onAddRelationship,
  person,
  personId: _personId,
  readOnly: _readOnly,
  relation,
  selected,
  ...props
}: FamilyCardProps<Profile>) {
  return (
    <article {...props} className="memoir-profile-card">
      <strong>{person.name}</strong>
      <small>{selected ? "selected" : relation.label}</small>
    </article>
  );
}

export function Page() {
  return (
    <FamilyTree
      profiles={profiles}
      relationships={relationships}
      rootProfileId="alex"
      renderProfileCard={(_profile, props) => <ProfileCard {...props} />}
    />
  );
}
```

You can also omit `renderProfileCard` to use the built-in default card:

```tsx
export function Page() {
  return <FamilyTree profiles={profiles} rootProfileId="alex" relationships={relationships} />;
}
```

## Styling

<div className="docs-api-band">
  <h3>Stable selectors, normal CSS</h3>
  <div className="docs-copy">The renderer emits data attributes for surfaces, cards, relation labels, selected state, and SVG edges so you can style the component without wrapping internals.</div>
</div>

Use normal CSS against stable data attributes. This is the flat black-shadow look used on the site:

```css
[data-family-card] {
  min-width: 128px;
  border: 2px solid currentColor;
  border-radius: 0;
  padding: 10px 12px;
  background: #fffdf4;
  box-shadow: 6px 6px 0 #030201;
}

[data-family-card][data-relation="self"] {
  background: #ec5a44;
}

[data-family-card][aria-selected="true"] {
  box-shadow: 8px 8px 0 #030201;
}

[data-family-edge] {
  color: #3b342e;
}
```
